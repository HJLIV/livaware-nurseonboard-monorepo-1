# Content Research Writer

Research topics thoroughly and produce well-structured, accurate content.

## When to Use

- Writing technical articles or blog posts
- Creating comprehensive documentation
- Researching a topic and summarizing findings
- User asks to "research and write about X"

## Process

### Step 1: Define Scope
- What is the topic?
- Who is the target audience?
- What's the desired length and depth?
- What format? (article, documentation, guide, report)

### Step 2: Research
- Search for authoritative sources on the topic
- Gather multiple perspectives
- Identify key concepts and terminology
- Note common misconceptions
- Collect examples and case studies

### Step 3: Outline
```markdown
# [Title]

## Introduction
- Hook / problem statement
- Why this matters
- What the reader will learn

## [Section 1: Background/Context]
- Key concepts
- Current state of things

## [Section 2: Main Content]
- Detailed explanation
- Examples
- Best practices

## [Section 3: Practical Application]
- Step-by-step guide
- Code examples (if technical)
- Common pitfalls

## Conclusion
- Summary of key points
- Next steps for the reader
- Additional resources
```

### Step 4: Write
- Write one section at a time
- Use clear, concise language
- Include examples for complex concepts
- Cite sources where appropriate
- Use headings, lists, and formatting for readability

### Step 5: Review
- Check factual accuracy
- Ensure logical flow between sections
- Remove redundancy
- Verify code examples work (if applicable)
- Proofread for grammar and clarity

## Writing Guidelines
- Lead with the most important information
- Use active voice
- Keep paragraphs short (3-5 sentences)
- Define jargon on first use
- Include practical examples
- Write at the audience's level (don't assume expertise)
