# Overdrive

Apply maximum effort and thoroughness to deliver exceptional quality work.

## When to Use

- User wants the highest quality implementation possible
- Critical feature that must be bulletproof
- Showcase or demo-worthy work
- User says "go all out", "no shortcuts", or "overdrive"

## Overdrive Checklist

### Implementation
- [ ] Every edge case identified and handled
- [ ] All error paths produce helpful messages
- [ ] Input validation is comprehensive
- [ ] Performance optimized for the expected scale
- [ ] Graceful degradation for all failure modes

### Testing
- [ ] Unit tests for all business logic
- [ ] Integration tests for all workflows
- [ ] Edge case tests (empty, null, boundary, large)
- [ ] Error scenario tests
- [ ] Performance/load tests if applicable

### Code Quality
- [ ] Clean, readable code throughout
- [ ] Consistent naming and conventions
- [ ] No dead code or debug statements
- [ ] Public APIs fully documented
- [ ] Complex logic commented with rationale

### User Experience
- [ ] Loading states for all async operations
- [ ] Error states with recovery suggestions
- [ ] Empty states with helpful guidance
- [ ] Smooth transitions between states
- [ ] Responsive across screen sizes
- [ ] Keyboard accessible
- [ ] Screen reader friendly

### Security
- [ ] Input sanitization complete
- [ ] Authentication/authorization verified
- [ ] No sensitive data exposure
- [ ] Rate limiting in place
- [ ] CORS properly configured

### Documentation
- [ ] README updated
- [ ] API documentation complete
- [ ] Architecture decisions documented
- [ ] Deployment instructions verified
- [ ] Troubleshooting guide included

## Process
1. Read and understand all requirements deeply
2. Design the solution architecture
3. Implement with full test coverage
4. Apply the entire checklist above
5. Review own code critically
6. Test every user-facing flow manually
7. Polish until satisfied with quality
