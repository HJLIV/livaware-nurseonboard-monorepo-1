# Quieter

Reduce visual noise and create calmer, more focused interfaces.

## When to Use

- UI feels cluttered or overwhelming
- Too many competing visual elements
- User asks for "quieter", "simpler", or "less busy" design
- Need to improve focus on key content

## Techniques

### Reduce Visual Weight
- Lower border opacity or remove borders
- Reduce shadow intensity
- Use lighter, more muted colors
- Decrease font weight for secondary text
- Remove decorative elements that don't serve function

### Increase Whitespace
- Add padding inside containers
- Add margin between sections
- Reduce items per row/screen
- Let content breathe
- Use line-height 1.5-1.7 for body text

### Simplify
- Remove or hide non-essential elements
- Use progressive disclosure (show on demand)
- Combine related controls
- Reduce the number of visible options
- Use sensible defaults to reduce form fields

### Mute Colors
- Replace saturated colors with muted tones
- Use monochromatic or analogous color schemes
- Reserve bright colors for important actions only
- Use neutral backgrounds (off-white, light gray)
- Reduce color variety (max 2-3 colors)

### Typography
- Use fewer font sizes (3-4 max)
- Use fewer font weights (2-3 max)
- Increase line height for readability
- Use consistent text alignment
- Remove uppercase styling except for small labels

## Application Order
1. Remove unnecessary elements (biggest impact)
2. Add whitespace (improves focus)
3. Mute colors (reduces stimulation)
4. Simplify typography (improves readability)
5. Refine details (subtle borders, shadows)
