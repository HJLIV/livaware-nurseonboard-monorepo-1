Context
Three separate repositories (nurse-onboard, clinical-skills-arcade, nurse-preboard) share an identical tech stack and design system but run as independent apps. The goal is to consolidate them into hjliv/livaware-nurseonboard-monorepo-1 as a unified platform with:

A single shared PostgreSQL database
A unified design system and UX/UI frontend
Proper monorepo structure enabling cross-app features

Implementation Steps
Phase 1: Monorepo Scaffold

Create root config files in livaware-nurseonboard-monorepo-1:

package.json with npm workspaces pointing to packages/* and apps/*
turbo.json for build orchestration (dev, build, lint tasks)
tsconfig.base.json with shared compiler options
Root .gitignore, .env.example



Phase 2: Shared Packages

Create packages/ui/ — Extract the shared design system:

Copy all shadcn/ui components (identical across repos) into packages/ui/components/
Move nurse-onboard's layout system (app-layout.tsx, sidebar-nav.tsx, portal-layout.tsx) into packages/ui/layout/
Move ThemeProvider and CSS variables into packages/ui/theme/
Shared tailwind.config.ts and components.json
Export everything via package.json


Create packages/db/ — Unified database layer:

Single Drizzle ORM connection
Schema files organized by domain:

schema/shared.ts — users, roles, auth, sessions (shared across apps)
schema/onboard.ts — candidates, compliance, documents, references
schema/arcade.ts — modules, scenarios, assessments, scores
schema/preboard.ts — preboard assessments, reports


All schemas in one PostgreSQL database (using table prefixes per app domain)


Create packages/shared/ — Common utilities:

Shared TypeScript types/interfaces
Auth utilities (login, session management)
API helper functions
Common constants



Phase 3: Migrate Apps

Migrate nurse-onboard → apps/nurse-onboard/:

Move client/src/pages/ and app-specific components
Move server/ (routes, AI modules, services)
Update imports to use @livaware/ui, @livaware/db, @livaware/shared
Remove duplicated UI components (now in packages/ui)


Migrate clinical-skills-arcade → apps/clinical-skills-arcade/:

Move client/src/pages/ (scenario-player, nurse-dashboard, admin pages)
Move server/ (routes, scoring engine, seed data)
Update imports to shared packages


Migrate nurse-preboard → apps/nurse-preboard/:

Move client/src/pages/ (assessment page)
Move server/ (AI, email, PDF, routes)
Update imports to shared packages



Phase 4: Unified Database

Consolidate database schemas:

Merge Drizzle schema files from all three repos
Create shared tables (users, roles) that all apps reference
Generate unified migrations
Create seed scripts that work with the unified schema



Phase 5: Unified Frontend Shell

Create shared app shell:

Unified navigation that links between all three app areas
Shared login/auth flow
Consistent sidebar navigation using nurse-onboard's layout patterns
Role-based routing (admin sees all apps, nurses see relevant sections)



Phase 6: Wiring & Polish

Dev tooling:

Root npm run dev starts all apps
Individual npm run dev --workspace=apps/nurse-onboard for single app
Shared environment variable handling
Turborepo caching for builds
